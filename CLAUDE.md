# Claude Instructions

Read [`AGENTS.md`](AGENTS.md) first. It is the source of truth for repository
workflow, architecture rules, testing policy, database changes, and safety.

To keep context small, read documentation progressively:

1. Use [`README.md`](README.md) for the project overview.
2. Use [`backend/README.md`](backend/README.md) when changing the backend.
3. Read only the README nearest to the module or directory being changed.
4. Use [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) only when cross-system
   architecture context is needed.

Do not duplicate rules from `AGENTS.md` into this file.
